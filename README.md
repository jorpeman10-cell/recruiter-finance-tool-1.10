# 猎头公司财务分析工具

一个专为猎头公司设计的财务分析Web应用，支持真实财务数据和内部核算版本两种模式。

## 主要功能

- **收入分析**：成单统计、顾问业绩、客户分析
- **成本分析**：真实运营成本或内部核算成本
- **利润分析**：基于实际回款和上年遗留回款计算
- **双模式切换**：真实数据（内部使用）/ 内部核算版本（给顾问展示）

## 核心概念

### 本年收入合计
**本年收入合计 = 本年实际回款 + 上年遗留回款**

- **本年实际回款**：本年成立单且在本年回款的金额
- **上年遗留回款**：去年或更早成立单但在本年回款的金额

### 两种分析模式

**1. 真实数据模式**
- 使用真实运营费用数据
- 顾问成本 = 真实底薪 + 提成
- 用于公司内部真实财务分析

**2. 内部核算版本（给顾问展示）**
- 顾问成本 = 基本月薪 × 3 × 1.2
- 3倍包括：工资、社保公积金、奖金、增值税、营运成本
- 不包含真实运营费用明细
- 用于给顾问展示的盈亏分析

## 快速开始

### 1. 安装依赖
```bash
pip install -r requirements.txt
```

### 2. 启动应用
```bash
streamlit run app.py
```

### 3. 准备数据

创建 CSV 文件，包含以下列：

| 列名 | 说明 | 必填 |
|------|------|------|
| deal_id | 成单编号 | ✅ |
| client_name | 客户名称 | ✅ |
| candidate_name | 候选人姓名 | ✅ |
| position | 职位 | ✅ |
| consultant | 顾问姓名 | ✅ |
| deal_date | 成单日期 (YYYY-MM-DD) | ✅ |
| fee_amount | 佣金金额 | ✅ |
| actual_payment | 本年实际回款 | ✅ |
| payment_status | 回款状态 | ✅ |
| prior_year_collection | 上年遗留回款 | ❌ |

### 4. CSV 示例

```csv
deal_id,client_name,candidate_name,position,consultant,deal_date,fee_amount,actual_payment,prior_year_collection,payment_status
D001,客户A,张三,经理,顾问A,2026-01-15,300000,300000,0,已回款
D002,客户B,李四,总监,顾问B,2026-02-20,500000,100000,400000,部分回款
D003,客户C,王五,工程师,顾问C,2026-03-10,200000,0,0,未回款
```

**数据说明：**
- D001：本年成单，本年回款30万
- D002：本年成单，本年回款10万 + 上年遗留回款40万 = 本年收入50万
- D003：本年成单，未回款

### 5. 上传数据

1. 在侧边栏点击"上传成单数据"
2. 选择 CSV 或 Excel 文件
3. 点击"加载上传的数据"
4. 在"数据概览"查看加载结果

## 常见问题

**Q: 本年收入显示为0？**
A: 请检查：
1. CSV文件是否包含 `fee_amount` 和 `actual_payment` 列
2. 数值是否为纯数字（不含逗号、货币符号）
3. 日期格式是否正确（YYYY-MM-DD）

**Q: 如何添加上年遗留回款？**
A: 在CSV中添加 `prior_year_collection` 列。

**Q: 内部核算版本需要费用数据吗？**
A: 不需要，成本通过公式自动计算。

**Q: 如何清空数据？**
A: 点击侧边栏"清空数据"按钮。
