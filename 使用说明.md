# 猎头财务分析工具 - 使用说明

## 核心功能

1. **实际收入分析** - 已成单和回款的统计分析
2. **预测收入分析** - 在途单的预测和未来收入预估
3. **双模式切换** - 真实数据（内部）/ 内部核算版本（给顾问展示）

---

## 一、实际收入数据（成单数据）

### 核心逻辑

**`actual_payment`** = 总回款（已包含本年回款 + 上年遗留回款）

**`prior_year_collection`** = 上年遗留回款（仅用于明细展示，已包含在 actual_payment 中）

**本年回款** = `actual_payment` - `prior_year_collection`

### CSV 文件格式

| 列名 | 说明 | 必填 |
|------|------|------|
| deal_id | 成单编号 | ✅ |
| client_name | 客户名称 | ✅ |
| candidate_name | 候选人姓名 | ✅ |
| position | 职位 | ✅ |
| consultant | 顾问姓名 | ✅ |
| deal_date | 成单日期 (YYYY-MM-DD) | ✅ |
| fee_amount | 佣金金额 | ✅ |
| actual_payment | **总回款**（本年+上年遗留） | ✅ |
| payment_status | 回款状态 | ✅ |
| prior_year_collection | 上年遗留回款（明细标注） | ❌ |

### 示例

```csv
deal_id,client_name,candidate_name,position,consultant,deal_date,fee_amount,actual_payment,prior_year_collection,payment_status
D001,客户A,张三,经理,顾问A,2026-01-15,300000,300000,0,已回款
D002,客户B,李四,总监,顾问B,2026-02-20,500000,500000,400000,已回款
D003,客户C,王五,工程师,顾问C,2026-03-10,200000,0,0,未回款
```

**数据说明：**
- D001：本年成单，本年回款30万，无上一年遗留
- D002：本年成单，总回款50万（其中本年10万 + 上年遗留40万）
- D003：本年成单，未回款

## 二、预测收入数据（在途单）

### 核心逻辑

**加权预测收入** = 预计佣金 × 成功率

用于评估在途单的未来收入潜力。

### CSV 文件格式

| 列名 | 说明 | 必填 |
|------|------|------|
| forecast_id | 预测单编号 | ✅ |
| client_name | 客户名称 | ✅ |
| position | 职位名称 | ✅ |
| consultant | 负责顾问 | ✅ |
| candidate_name | 候选人姓名 | ❌ |
| estimated_salary | 预计年薪 | ❌ |
| fee_rate | 费率(%) | ❌ |
| estimated_fee | 预计佣金 | ❌ |
| success_rate | 成功率(%) | ✅ |
| stage | 阶段 | ❌ |
| expected_close_date | 预计成交日期 | ❌ |
| note | 备注 | ❌ |

**说明：**
- `estimated_fee` 和 `estimated_salary`+`fee_rate` 二选一
- 如果提供 `estimated_fee`，则直接使用
- 否则计算：`estimated_salary` × `fee_rate` / 100
- `stage` 建议值：初期接触、推荐简历、面试中、offer谈判、待发offer

### 示例

```csv
forecast_id,client_name,position,consultant,candidate_name,estimated_salary,fee_rate,success_rate,stage,expected_close_date
F001,阿里巴巴,Java架构师,顾问A,候选人A,800000,20,70,面试中,2026-04-15
F002,腾讯,产品经理,顾问B,候选人B,600000,20,50,初期接触,2026-05-01
F003,字节跳动,算法工程师,顾问A,候选人C,1000000,22,80,offer谈判,2026-03-30
```

**分析结果：**
- F001：预计佣金16万 × 70%成功率 = 加权收入11.2万
- F002：预计佣金12万 × 50%成功率 = 加权收入6万
- F003：预计佣金22万 × 80%成功率 = 加权收入17.6万
- **合计**：预计佣金50万，加权预测收入34.8万

## 三、使用步骤

### 1. 启动应用
```bash
streamlit run app.py
```

### 2. 上传实际收入数据
- 在侧边栏点击"上传成单数据"
- 选择 CSV 或 Excel 文件
- 点击"加载上传的数据"

### 3. 上传预测数据（可选）
- 在侧边栏点击"上传预测/在途单数据"
- 选择 CSV 或 Excel 文件
- 点击"加载上传的数据"

### 4. 查看分析
- **总览** - 关键指标汇总
- **收入分析** - 实际收入分析
- **成本分析** - 成本结构分析
- **利润分析** - 利润计算
- **KPI仪表板** - 业务指标
- **预测分析** - 在途单预测（第6个标签页）

## 四、两种分析模式

### 真实数据模式（内部使用）
- 使用真实运营费用数据
- 顾问成本 = 真实底薪 + 提成
- 需要上传费用数据（可选）

### 内部核算版本（给顾问展示）
- 顾问成本 = 基本月薪 × 3 × 1.2
- 3倍包括：工资、社保公积金、奖金、增值税、营运成本
- 不需要上传费用数据
- 避免真实数据泄露

## 六、常见问题

**Q: actual_payment 和 prior_year_collection 的关系？**
A: `actual_payment` 是总回款，已经包含了 `prior_year_collection`。`prior_year_collection` 只是用来标注哪些回款是来自上年的。

**Q: 预测数据中的 estimated_fee 和 estimated_salary 怎么填？**
A: 二选一：
- 如果知道预计佣金，直接填 `estimated_fee`
- 否则填 `estimated_salary` 和 `fee_rate`，系统会自动计算

**Q: 成功率怎么设置？**
A: 根据阶段设置参考值：
- 初期接触：20-30%
- 推荐简历：30-50%
- 面试中：50-70%
- offer谈判：70-85%
- 待发offer：85-95%

**Q: 可以同时上传实际数据和预测数据吗？**
A: 可以，两者是独立的。实际数据用于统计已完成的业绩，预测数据用于评估未来的收入潜力。
