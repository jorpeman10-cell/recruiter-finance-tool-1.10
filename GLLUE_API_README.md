# Gllue API 同步功能使用说明

## 功能简介

通过 Gllue（谷露）系统 API，自动同步 Offer、入职等数据到财务分析工具，省去手动上传 Excel 的麻烦。

## 前置条件

### 1. 获取 Gllue API 密钥

联系谷露客服或你的系统管理员，申请 API 访问权限。你需要：
- API Key（密钥）
- Gllue 系统域名（如 `yourcompany.gllue.com`）

### 2. 权限要求

确保 API 账号有以下权限：
- 查看 Offer 列表 (`/rest/offersign/`)
- 查看入职列表 (`/rest/onboard/`)
- 查看职位列表 (`/rest/joborder/`)
- 查看用户列表 (`/rest/user/`)

## 使用步骤

### 第一步：配置 API 连接

1. 在侧边栏找到 **"🔄 Gllue API 同步"** 区域
2. 点击 **"配置谷露 API 连接"** 展开配置面板
3. 填写以下信息：
   - **Gllue 域名**：你的谷露系统域名（如 `abc.gllue.com`，不需要 https://）
   - **API 密钥**：从谷露获取的 API Key
   - **开始日期**：同步此日期之后的数据（建议设为一年前）
   - **结束日期**：同步此日期之前的数据（建议设为今天）
4. 点击 **"💾 保存配置"**

### 第二步：测试连接

点击 **"🔌 测试连接"** 按钮，系统会尝试连接 Gllue API：
- ✅ 连接成功：会显示获取到的用户数量
- ❌ 连接失败：会显示错误信息，请检查域名和密钥

### 第三步：同步数据

点击 **"🚀 同步数据"** 按钮，系统会：
1. 从 Gllue 获取 Offer 列表（作为成单数据）
2. 获取入职列表（补充入职日期等信息）
3. 自动转换为财务工具的数据格式
4. 加载到分析器中

同步完成后，你就可以像使用手动上传的数据一样进行财务分析了。

## 数据映射说明

### Gllue 字段 → 财务工具字段

| Gllue 字段 | 财务工具字段 | 说明 |
|------------|--------------|------|
| `jobsubmission__joborder__id` | 职位ID | 职位编号 |
| `jobsubmission__joborder__bu____name__` | 客户名称 | 用人部门/客户 |
| `jobsubmission__joborder__jobTitle` | 职位名称 | 职位标题 |
| `jobsubmission__joborder__joborderuser_set__user____name__` | 顾问 | 负责顾问 |
| `signDate` | 成单日期 | Offer 签署日期 |
| `onboardDate` | 入职日期 | 预计/实际入职日期 |
| `annualSalary` | 年薪 | 候选人年薪 |
| 固定值 20 | 费率% | 默认按 20% 计算 |
| `annualSalary × 20%` | 佣金总额 | 应收佣金 |

### 注意：需要手动补充的字段

以下字段 API 无法自动获取，需要你在 Gllue 中维护或同步后手动补充：

1. **实际回款金额** - 需要从财务系统获取
2. **回款日期** - 实际到账日期
3. **发票日期** - 开票日期
4. **上年遗留回款** - 历史遗留数据

## 常见问题

### Q1: 连接测试失败怎么办？

**检查清单：**
- [ ] 域名是否正确（不需要 `https://` 前缀）
- [ ] API 密钥是否有效（联系谷露客服确认）
- [ ] 网络是否可以访问 Gllue 服务器
- [ ] API 账号是否有足够权限

### Q2: 同步后数据不完整？

**可能原因：**
- 日期范围设置太小 → 调大开始日期范围
- Gllue 中某些字段为空 → 在 Gllue 中完善职位信息
- API 权限不足 → 联系管理员开通权限

### Q3: 如何定期自动同步？

目前需要手动点击同步。如需自动同步，可以：
1. 使用 Windows 计划任务 + Python 脚本调用同步 API
2. 或者每天手动同步一次（建议上班时同步）

### Q4: 同步会覆盖现有数据吗？

是的，每次同步会清空现有数据并重新加载。如需保留历史数据，建议：
1. 先导出当前数据备份
2. 或者使用缓存功能（后续版本支持）

## 技术说明

### API 调用频率限制

根据 Gllue 文档，接口有访问频率限制。同步时会自动分页获取，避免触发限制。

### 数据安全

- API 密钥保存在内存中，页面刷新后需要重新输入
- 不会将密钥保存到文件或数据库
- 所有数据传输通过 HTTPS 加密

### 支持的 Gllue 版本

基于 Gllue 通用接口文档 V0.7.2 开发，兼容：
- Gllue 标准版
- Gllue 企业版

## 故障排查

### 查看错误日志

如果同步失败，错误信息会显示在页面上。常见错误：

| 错误信息 | 原因 | 解决方案 |
|----------|------|----------|
| `Connection refused` | 域名错误或网络不通 | 检查域名和网络 |
| `401 Unauthorized` | API 密钥无效 | 联系谷露确认密钥 |
| `403 Forbidden` | 权限不足 | 联系管理员开通权限 |
| `KeyError: 'xxx'` | 字段映射错误 | 检查 Gllue 字段配置 |

### 联系支持

如遇到无法解决的问题：
1. 记录错误信息和操作步骤
2. 联系谷露技术支持
3. 或提交 Issue 到财务工具项目
